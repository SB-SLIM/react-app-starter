# Commit Messages — sb-codex monorepo

## Format

```
<type>(<scope>): <subject>

[optional body]

[optional footer]
```

- **Subject**: imperative mood, lowercase, no period, ≤ 72 chars.
- **Body**: explain *why*, not *what*. Wrap at 100 chars.
- **Footer**: `BREAKING CHANGE: <description>` or `Closes #<issue>`.

---

## Types

| Type | When to use |
|---|---|
| `feat` | New capability visible to a user or consumer of a package |
| `fix` | Bug fix — something was broken and now it isn't |
| `perf` | Performance improvement with no behaviour change |
| `refactor` | Code restructure with no behaviour or API change |
| `test` | Adding or fixing tests only |
| `docs` | Documentation only (CLAUDE.md, README, architecture.md, ADRs) |
| `chore` | Tooling, config, CI, deps, scripts — no production code |
| `build` | Changes to build system (turbo.json, tsup, Vite config, Dockerfiles) |
| `ci` | GitHub Actions workflow changes |
| `revert` | Reverts a previous commit (auto-generated by git) |

**Never use `feat` for internal refactors.** If the public API of a package didn't change, it's `refactor`.

## Scopes

Scopes map directly to the monorepo graph. Use the shortest unambiguous scope.

### Packages
| Scope | Maps to |
|---|---|
| `core` | `packages/core` |
| `ui` | `packages/ui-components` |
| `db` | `packages/db` |
| `auth` | `packages/auth` |
| `acl` | `packages/acl` |
| `api` | `packages/api-contracts` |
| `jobs` | `packages/jobs` |
| `config` | `packages/config` |

### Apps
| Scope | Maps to |
|---|---|
| `admin` | `apps/admin` |
| `server` | `apps/server` |
| `web` | `apps/web` |
| `superadmin` | `apps/superadmin` |
| `e2e` | `apps/e2e` |

### Infrastructure / tooling
| Scope | Maps to |
|---|---|
| `infra` | `infra/` (Docker, Traefik, compose) |
| `ci` | `.github/workflows/` |
| `deps` | dependency updates (Dependabot / manual upgrades) |
| `release` | changeset publish, version bumps |
| `.claude` | `.claude/` harness config (settings, skills, agents, hooks) |

### Cross-cutting
Use no scope (or `workspace`) when the change spans multiple packages/apps with no single owner.

## Examples

```
feat(acl): add manager:export permission to ROLE_PERMISSIONS

fix(db): add missing workspace_id index on client table

perf(server): replace full-table scan in clients.list with cursor pagination

refactor(auth): extract token-refresh logic into standalone helper

test(acl): add requirePermission unit tests for all roles

docs(architecture): document RLS connection role separation

chore(deps): upgrade drizzle-orm to 0.44.1

build(infra): add non-root USER directive to all Dockerfiles

ci: add pnpm audit --audit-level=high to CI workflow

feat(admin)!: replace useRole with usePermission in all feature guards

BREAKING CHANGE: useRole is removed from @sb-codex/acl/client. Replace
all calls with usePermission('resource:action') per the new permission model.
```

## Common mistakes to avoid

| ❌ Wrong | ✅ Correct |
|---|---|
| `fix: various fixes` | `fix(server): handle null workspace in enforceWorkspace middleware` |
| `feat: updated the auth` | `feat(auth): add signInWithGitHub to createSbAuthClient` |
| `WIP` | Don't commit WIP — stash instead |
| `feat(packages/db): ...` | `feat(db): ...` — scope is the short name, not the path |
| `Feat(Admin): Added thing` | `feat(admin): add thing` — lowercase, imperative |
| `chore: update deps` | `chore(deps): upgrade @trpc/server to 11.1.0` |

## Breaking changes

A `!` after the type+scope marks a breaking change and **must** include a `BREAKING CHANGE:` footer.

```
feat(api)!: remove deprecated healthRouter.ping procedure

BREAKING CHANGE: healthRouter.ping was removed. Use healthRouter.check instead.
Clients must update their tRPC callers before upgrading apps/server.
```

Breaking changes trigger a **major** version bump in changesets.
